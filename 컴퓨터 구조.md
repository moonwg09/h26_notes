- 1:N : 여러 개의 사용자 스레드가 하나의 커널 스레드를 사용  => 멀티코어 환경에서 잘 안쓴다

1. cpu 스케줄링의 개요

1-1cpu 스케줄링 : 운영체제가 프로세스들에게 공정하게 cpu자원 분배

- 단계별 스케줄링 : 작업대기 -> 보류 프로세스 -> 활성 프로세스 -> 실행프로세스 -> 작업종료
고수준, 중수준, 저수준 스케줄링
디스패치:

- 선점형 스케줄링, 비선점형 스케줄링
선점형 : 스케줄러가 실행중인 프로세스 cpu를 강제로 빼앗아 다른 프로세스에 할당할 수 있는 스케줄링 방식(시분할 시스템, 인터럽트)
비선점형 : 프로세스가 cpu를 사용하면 작업이 끝날 때까지 빼앗을 수 없는 스케줄링 방식(일괄 작업 시스템)

선점형, 비선점형 비교

장점 : 빠른 응답을 요구하는 시스템에 적합, 문맥교환과 같은 부가적인 작업이 없다.
단점 : 문맥 교환의 오버헤드가 많다, 실행 시간이 긴 작업 때문에 실행시간이 작은 작업이 오래 기다려서 처리율이 떨어진다.
우선순위 : 높다, 낮다

-전면: 그래픽 사용자 인터페이스 환경에서 맨 앞에서 실행되는 프로세스(상호작용 프로세스)
-후면: 사용자와의 상호작용이 없는 일괄 작업 프로세스(압축, 디스크 백업, 자동 업데이트)


프로세스의 우선순위를 판단하는 기준
커널 <-> 사용자
전면 <-> 후면
대화형 <-> 일괄작업
입출력 집중 <->  cpu 집중

고정 우선순위와 변동 우선순위

준비 큐와 대기 큐

준비큐는 우선순위에 따라 여러 줄로 구성된다

2. cpu 스케줄링 알고리즘

2-1 알고리즘 종류와 평가

cpu 스케줄링 알고리즘 종류(구별할수 잇으면 된다)
-비선점형
 
-선점형

비선점형, 선점형
우선순위 미적용
선입선출(비선형), 라운드 로빈(선형)

우선순위 적용
최단작업우선, 최고 응답 우선(비선형)    	최소 잔여시간, 다단계 큐, 다단계 피드백 큐(선형)


평가 기준
-cpu 사용률
-처리량
-대기 시간
-응답 시간
-반환 시간


2-2 선입선출 스케줄링

일괄작업 시스템
콘보이 효과 : 짧은 프로세스가 도착해도 오랫동안 기다려야하는 문제가 발생할 수 있다

2-3 최단 작업 우선 스케줄링

실행 시간이 짧은 작업부터 cpu를 할당하는 스케줄링
종료시간 예측의 어려움
공평성 위배
기아 현상 : 특정 프로세스가 cpu나 자원을 계속 할당받지 못하는 상태
에이징(기아현상의 해결책) : 프로세스가 양보할 수 있는 상한선을 정하는 기법

2-4 최고 응답 우선 스케줄링(HRN)
기아현상을 완화하는 비선점형 알고리즘
대기 시간과 실행 시간을 모두 고려하여 우선순위를 결정

2-5 라운드 로빈 스케줄링(RR)
모든 프로세스가 일정한 시간(타임 슬라이스) 동안만 cpu를 사용하는 방식
순환 순서 방식
작업이 끝나지 않으면 준비 큐의 맨뒤로 이동

2-6 최소 잔여 시간 스케줄링(SRT)
남아있는 작업 시간이 더 적은 프로세스에 먼저 cpu 할당
최단 작업 우선 스케줄링을 선점형으로 변환한 것

2-7 다단계 큐(MLQ) 스케줄링
단점 : 하위 큐에 있는 우선순위가 낮은 프로세스의 실행이 계속 지연
각 큐는 라운드 로빈 방식을 운영
작업 효율성 증가

2-8 다단계 피드백 큐(MLFQ) 스케줄링
우선순위가 동적으로 변하는 변동 우선순위 방식
다단계 큐 스케줄링의 단점을 완화하는 방식
